Development

In development, Ali uses the webpack-dev-server to bundle and serve assets from memory. Inline sourcemaps are enabled (no "waiting for sourcemaps to load" message in Chrome), and optimization and minification is skipped. No DLLs are used.

To use the development server, simply run npm start or yarn start and point your browser to localhost:8080. If you need to proxy api requests to a backend server, see tasks/serve.js and search for the commented proxy block.

If you would like to do a development build, but write it to disk rather than use the dev server, use npm run dev or yarn dev.
