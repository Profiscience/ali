Ali

An opinionated KnockoutJS meta-framework for building modern SPAs

What is it?

Ali is a meta-framework — a loose collection of packages and some minimal boilerplate — used at Profisciencē to build enterprise KnockoutJS SPAs with a strong focus on user experience without developer buy-in.

Declarative and highly-modular, Ali is easy to test and scalable to hundreds of pages.

Built on top of:
  • KnockoutJS
  • ko-component-router
  • webpack

Including goodies from:
  • knockout-punches
  • ko-contrib-utils
  • ko-contrib-fns

Supporting the following OOTB:
  • Incremental Asset Loading
  • Simple Async with Automatic Loading Animation
  • Optimized Production Builds with Long-Term Caching
  • ES2015/17 Support
  • SCSS Support
  • Smiles

How Do I Use It?

Clone or fork the repo, delete the views — everything in src other than web_modules — for this site and anything you don't need in web_modules, and start coding.

Why not X!?!?

Investment

When you have half a million lines of code written using something, hopping onto the next framework isn't always an option. For us, that something was Knockout, and over the years we've learned how to make pretty darn powerful apps with it using the patterns seen here.

Browser Support

Trust me, we all celebrated the day IE8 fell into obsolescence. Still, Knockout offers the best browser support out there and for some of us, that is important.
