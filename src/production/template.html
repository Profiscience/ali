Production

NOTE Production builds are turned on by setting NODE_ENV to production. Using the scripts fields in package.json does this automatically.

In production, vendor code and shared code in web_modules are built into vendor.[hash].js and shared.[hash].js "DLL"s. These will both be loaded on page load before the app entry. This decreases production build time and enables long term caching. In my experience, this approach is much preferable to using the CommonsChunkPlugin.

JavaScripts, HTML, and SCSS will all be minified, and gzipped assets will be produced alongside the regular versions.

To build for production, use npm run prod or yarn prod.
