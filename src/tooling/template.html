Tooling

Ali uses Fly for task orchestration and webpack for bundling the app and as a development webserver.

Why Fly instead of something more popular like gulp? Fly is small, fast, and does just enough, leaving most of the real work to webpack.

Any user-facing commands are exposed via the scripts property in package.json, and can be called with yarn or npm.

When using npm, tasks are ran using npm run <task>

When using yarn, run may be omitted; i.e. yarn <task>
